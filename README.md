## About The Project

Log file ingestor with UI for viewing data

Features:
* Ingest bulk logs on API ENDPOINT ```http://localhost:3000``` through POST request.
* Various filters with regex full-text search.
* Timestamp range search.
* Fast, scalable, reliable. :smile:

### Installation

This is how you run the log ingestor:

1. Clone the repo
2. Run Elasticsearch cluster with 3 nodes:
   ```sh
   cd elasticsearch
   sudo docker compose up -d
   ```
   NOTE: wait for some time for elasticsearch to finish setup (5 mins).
3. Run the backend:
   ```js
   cd backend
   npm install
   node index.js
   ```

4. Ingesst logs using POST request on ```http://localhost:3000```
   Resquest JSON Body:
   ```js
   [
     {
	"level": "error",
	"message": "Failed to connect to DB",
    "resourceId": "server-1234",
	"timestamp": "2023-09-15T08:00:00Z",
	"traceId": "abc-xyz-123",
    "spanId": "span-456",
    "commit": "5e5342f",
    "metadata": {
        "parentResourceId": "server-0987"
    }
    },
   	{
	"level": "error",
	"message": "Failed to connect to DB",
    "resourceId": "server-1234",
	"timestamp": "2023-09-15T08:00:00Z",
	"traceId": "abc-xyz-123",
    "spanId": "span-456",
    "commit": "5e5342f",
    "metadata": {
        "parentResourceId": "server-0987"
    }
	}
   ]
   ```

5. Run frontend:
   ```sh
   cd frontend
   npm install
   npm start
   ````

6. Visit ```http://localhost:5299``` to run the frontend UI.

7. Filter, view and regex logs.
